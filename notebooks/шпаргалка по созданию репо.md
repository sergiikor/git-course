# 🚀 Полная шпаргалка: Создание Git-проектов (2 способа)

В зависимости от того, начинаешь ли ты проект с чистого листа или у тебя уже есть папка с кодом на компьютере, выбери подходящий сценарий.

---

## 🟢 Вариант 1: Сначала GitHub (Remote-first / Рекомендуемый)
*Этот способ самый простой: связь с сервером настраивается автоматически.*

1. **На GitHub:**
   * Нажми **"New repository"**.
   * Укажи имя проекта.
   * Поставь галочку **"Add a README file"**.
   * Выбери шаблон **.gitignore** (например, для Python).
   * Нажми **"Create repository"**.

2. **В терминале (на компьютере):**
   * `cd ~/Desktop` (перейди в нужную папку)
   * `git clone https://github.com/твой_логин/название_проекта.git`
   * `cd название_проекта`
   * `git status`

---

## 🔵 Вариант 2: Сначала локальный код (Если папка уже есть)
*Используй этот способ, если код уже написан и его нужно привязать к GitHub.*

1. **На GitHub:**
   * Создай новый репозиторий.
   * **ВАЖНО:** Не ставь галочки на README, .gitignore и лицензии. Репозиторий должен быть абсолютно пустым.

2. **В терминале (внутри папки с твоим кодом):**
   * `git init` (инициализация)
   * `git add .` (подготовка файлов)
   * `git commit -m "Initial commit"`
   * `git branch -M main` (называем главную ветку)
   * `git remote add origin ССЫЛКА_НА_РЕПОЗИТОРИЙ`
   * `git push -u origin main` (первая отправка)

---

## 💡 Базовые команды на каждый день

| Команда | Что делает |
| :--- | :--- |
| git pull | Забирает изменения с сервера |
| git status | Проверяет состояние файлов |
| git add . | Добавляет всё в очередь на сохранение |
| git commit -m "текст" | Сохраняет с описанием |
| git push | Отправляет на GitHub |
| git checkout -b имя | Создает новую ветку |
