# 🛠 Git Magic: Шпаргалка по исправлению истории

### 1. Исправить последний коммит (Amend)
Если сделал опечатку или забыл добавить файл в последний коммит.
* **Правка кода:** `git add .` -> `git commit --amend --no-edit`
* **Правка сообщения:** `git commit --amend -m "Новое сообщение"`

---

### 2. Интерактивный Rebase (Хирургия)
Позволяет менять коммиты в прошлом.
`git rebase -i HEAD~n` (где `n` — количество коммитов для просмотра).

**Команды внутри редактора (заменяем `pick` на нужную букву):**
* `p` (pick) — оставить как есть.
* `r` (reword) — изменить только текст сообщения.
* `e` (edit) — остановиться и изменить код внутри коммита.
* `s` (squash) — склеить коммит с тем, что выше.
* `d` (drop) — удалить коммит.

---

### 3. Склеивание (Squash) — алгоритм
1. Вызвать `git rebase -i HEAD~[число]`.
2. Оставить верхний коммит `pick`, остальным прописать `s`.
3. Сохранить и закрыть вкладку.
4. В новом окне удалить старые сообщения и написать одно итоговое.
5. Сохранить и закрыть.



---

### 4. Экстренные ситуации
* **Запутался в конфликтах:** `git rebase --abort` (вернет всё как было до начала ребейза).
* **Разрулил конфликт:** `git add .` -> `git rebase --continue`.
* **Сделал ребейз и нужно отправить на сервер:** `git push origin [имя_ветки] --force` (Внимание: только для личных веток!).

---

> 💡 **Правило:** Кодируй "грязно" (много мелких коммитов), а пушь "чисто" (один логический коммит через squash).
